# الأسئلة الشائعة (FAQ)

## عام

### ما هي عُقدة؟

شبكة عُقدة هي بروتوكول شبكة mesh لامركزية محسّن للأداء العملي. تمكن من الاتصال المباشر بين الأقران مع تشفير من طرف إلى طرف، وإعدادات دنيا، واستقلالية عن البنية التحتية المركزية.

### كيف تختلف عُقدة عن Yggdrasil؟

عُقدة مبنية على شبكة Yggdrasil v0.5 مع تحسينات أداء كبيرة:

- **إنشاء اتصال أسرع** (20-40ms مقابل 50-100ms)
- **تقليل المهلات** لتحسين الاستجابة
- **تخزين DNS مؤقت** لتقليل عبء البحث
- **خوارزمية backoff تكيفية** لإعادة الاتصال الأسرع

عقد عُقدة **متوافقة 100% مع البروتوكول** مع عقد Yggdrasil v0.5 ويمكنها الاتصال معها بسلاسة. راجع [../ATTRIBUTION.md](../ATTRIBUTION.md) للتفاصيل.

### هل عُقدة جاهزة للإنتاج؟

**نعم، للأحمال غير الحرجة.** عُقدة موثوقة لـ:
- الشبكات المجتمعية
- سيناريوهات الحوسبة الطرفية
- التطوير والبحث
- النشر التجريبي

**غير موصى بها** لـ:
- الأنظمة الحرجة
- تطبيقات سلامة الحياة
- بيئات الأمان العالية (بدون تدقيق مستقل)

### ماذا تعني "عُقدة"؟

"عُقدة" (عُقدة) تعني بالعربية "عُقدة" أو "عُقدة"، تمثل الطبيعة المترابطة لشبكات mesh.

## تقني

### هل يمكن لعقد عُقدة الاتصال مع عقد Yggdrasil؟

**نعم!** عُقدة متوافقة مع البروتوكول مع Yggdrasil v0.5. يمكن للعقد من كلا الشبكتين الاتصال والتواصل بسلاسة.

### هل توفر عُقدة إخفاء الهوية؟

**لا.** عُقدة لا توفر إخفاء الهوية:

- يمكن للأقران المباشرين رؤية عنوان IP الخاص بك
- قد تكون أنماط حركة المرور قابلة للملاحظة
- لا توجد حماية ضد تحليل حركة المرور

إذا كنت تحتاج إخفاء الهوية، استخدم **Tor** أو **I2P** بدلاً من ذلك.

### كيف أُعد جدار الحماية؟

عقد عُقدة قابلة للوصول عالمياً افتراضياً. **موصى به بشدة**: إعداد قواعد جدار حماية IPv6.

**Linux (iptables/ip6tables):**
```bash
# حظر جميع الواردات على واجهة عُقدة
ip6tables -A INPUT -i uqda0 -j DROP

# السماح بالاتصالات المؤسسة
ip6tables -A INPUT -i uqda0 -m state --state ESTABLISHED,RELATED -j ACCEPT

# السماح بخدمات محددة (مثال: SSH)
ip6tables -A INPUT -i uqda0 -p tcp --dport 22 -j ACCEPT
```

**Windows (PowerShell):**
```powershell
# حظر جميع الواردات على واجهة عُقدة
New-NetFirewallRule -DisplayName "Block عُقدة Inbound" `
    -Direction Inbound -InterfaceAlias "Uqda" -Action Block

# السماح بخدمة محددة (مثال: SSH)
New-NetFirewallRule -DisplayName "Allow SSH on عُقدة" `
    -Direction Inbound -InterfaceAlias "Uqda" -Protocol TCP -LocalPort 22 -Action Allow
```

**macOS (pfctl):**
```bash
# إضافة إلى /etc/pf.conf:
block in on uqda0 all
pass in on uqda0 proto tcp from any to any port 22
```

راجع [../SECURITY.md](../SECURITY.md) لمزيد من التفاصيل.

### كيف يعمل نظام العناوين؟

كل عُقدة لديها **عنوان IPv6 دائم** مشتق من مفتاحها العام:

```
عنوان IPv6 = truncate(hash(public_key))
```

هذا يعني:
- عنوانك **لا يتغير أبداً** (ما لم تقم بإعادة توليد المفاتيح)
- العنوان **مستقل عن الموقع** (يعمل في أي مكان)
- **لا حاجة لتخصيص مركزي** (لا DHCP، لا سجلات)

العناوين تستخدم نطاق IPv6 `0200::/7` (مهمل من IETF، لا تعارض مع ULA).

### ما بروتوكولات النقل المدعومة؟

- **TCP** (مع TLS اختياري)
- **QUIC** (تجريبي)
- **WebSocket/WebSocket Secure** (لتوافق المتصفح)
- **نقل IPv4 أو IPv6**

### هل تعمل عُقدة خلف NAT؟

**نعم!** عُقدة تتعامل مع بيئات NAT بسلاسة:

- اتصالات الأقران **ثنائية الاتجاه** بمجرد إنشائها
- لا حاجة لإعادة توجيه المنافذ للاتصالات الصادرة
- العقد خلف NAT تبقى قابلة للوصول بالكامل داخل شبكة عُقدة
- فعالة حتى مع NAT على مستوى الناقل (CGNAT)

## الأداء

### لماذا عُقدة أسرع؟

عُقدة تتضمن عدة تحسينات للأداء:

1. **تقليل المهلات**:
   - المصافحة: 5s (مقابل 6s)
   - اتصال TCP: 3s (مقابل 5s)
   - WebSocket: 5s (مقابل 10s)

2. **تخزين DNS مؤقت**:
   - عمليات البحث الناجحة مخزنة لمدة 5 دقائق
   - عمليات البحث الفاشلة مخزنة لمدة 30 ثانية
   - يقلل زمن الاستجابة بمقدار 10-30ms لكل اتصال

3. **Backoff تكيفي**:
   - المحاولة الأولى: 100ms (مقابل 1s)
   - إعادة اتصال أسرع بعد الأعطال المؤقتة

راجع [WHITEPAPER.md](WHITEPAPER.md) للتفاصيل التقنية الكاملة.

### ما مقدار تحسين زمن الاستجابة؟

في السيناريوهات النموذجية:
- **إنشاء الاتصال**: 20-40ms (مقابل 50-100ms الأساسي)
- **زمن الاستجابة من طرف إلى طرف**: تحسين 20-50ms
- **وقت إعادة الاتصال**: أسرع بمقدار 500-900ms

في شبكة من 10 قفزات مع 3 أعطال مؤقتة في الساعة، تقلل هذه التحسينات متوسط زمن الاستجابة من طرف إلى طرف بمقدار **35-55ms**.

### كم عرض النطاق الذي تستخدمه عُقدة؟

عبء البروتوكول ضئيل:
- تبادلات مرشح Bloom للتوجيه
- رسائل صيانة الشجرة الممتدة
- إعادة توجيه البيانات متناسبة مع موقع الشبكة

العقد مع اتصالات أقران متعددة ستعيد توجيه حركة المرور للآخرين (هذه هي طريقة عمل شبكات mesh).

## الأمان

### هل تم تدقيق عُقدة؟

**ليس بعد.** عُقدة لم تخضع لتدقيق أمني مستقل.

الكود يرث تطبيقات التشفير من:
- مكتبة Go القياسية (مدققة)
- شبكة Yggdrasil (مراجعة مجتمعية منذ 2017)

**التوصيات**:
- لا تستخدم للتطبيقات عالية الأمان بدون مراجعة مستقلة
- افترض وجود ثغرات غير مكتشفة
- الدفاع المتعدد الطبقات: استخدم التشفير على مستوى التطبيق للبيانات الحساسة

**مخطط**: تدقيق ممول من المجتمع مجدول لـ Q2 2026.

### هل من الآمن استخدامها؟

**نعم، مع الاحتياطات المناسبة:**

1. **إعداد جدران الحماية** (انظر أعلاه)
2. **لا تعرض الخدمات الحساسة** بدون أمان إضافي
3. **استخدم التشفير على مستوى التطبيق** للبيانات عالية الحساسية
4. **راقب عقدتك** لحركة مرور غير متوقعة

راجع [../SECURITY.md](../SECURITY.md) لإرشادات الأمان التفصيلية.

## حالات الاستخدام

### ما الذي يمكنني استخدام عُقدة له؟

- **شبكات mesh المجتمعية** - الأحياء تبني بنية تحتية مستقلة
- **الحوسبة الطرفية** - اتصال مباشر بين الأجهزة
- **الشبكات الطارئة** - نشر سريع بدون بنية تحتية
- **البحث والتطوير** - تجربة البروتوكولات
- **التطبيقات اللامركزية** - خدمات من نظير إلى نظير

### ما الذي لا يجب أن أستخدم عُقدة له؟

- **إخفاء الهوية** - استخدم Tor/I2P بدلاً من ذلك
- **استبدال VPN** - عُقدة غير مصممة كـ VPN تقليدي
- **الأنظمة الحرجة** - غير جاهزة للإنتاج للأنظمة الحرجة بعد
- **بيئات الأمان العالية** - بدون تدقيق مستقل

## استكشاف الأخطاء

### عقدتي لا تتصل بالأقران

1. **تحقق من قواعد جدار الحماية** - تأكد من أنك لا تحظر الاتصالات الصادرة
2. **تحقق من عناوين الأقران** - تأكد من صحة عناوين URL للأقران
3. **تحقق من اتصال الشبكة** - هل يمكنك الوصول إلى عنوان IP للقرين؟
4. **راجع السجلات** - ابحث عن رسائل الخطأ

### الاتصال بطيء

1. **تحقق من مسافة القرين** - اختر أقراناً قريبين جغرافياً
2. **راقب جدول التوجيه** - استخدم `uqdactl getPeers` لرؤية الاتصالات النشطة
3. **راجع مقاييس الأداء** - راجع [WHITEPAPER.md](WHITEPAPER.md) للتفاصيل التقنية

### كيف أضيف/أزيل أقراناً؟

**باستخدام uqdactl:**
```bash
# إضافة قرين
uqdactl addPeer tcp://peer.example.com:12345

# إزالة قرين
uqdactl removePeer tcp://peer.example.com:12345

# عرض الأقران
uqdactl getPeers
```

**باستخدام ملف الإعدادات:**
قم بتعديل `uqda.conf` وأضف/أزل إدخالات من مصفوفة `Peers`.
